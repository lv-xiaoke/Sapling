# 儿童教育 AI 问答 + 视频生成 — 设计方案

> 日期：2026-05-15
> 状态：设计阶段，尚未实现

---

## 1. 项目定位

搭建一个内部使用的 AI 工具，围绕**儿童教育**方向，提供两类能力：

1. **教育问答**：家长上传育儿资料后，基于资料进行自然语言问答
2. **个性化视频生成**：为 0-3 岁幼儿生成有声绘本风格的定制视频

当前阶段：先服务一个有两个幼儿（2 岁+、不到 1 岁）的家庭，跑通验证后再考虑扩展。

---

## 2. 整体架构

两个子系统共享同一套知识库，走不同的输出管线：

```
┌─────────────────────────────────────────────────┐
│                  知识库（育儿书 PDF）              │
│              Embedding + 向量检索                 │
├──────────────────────┬──────────────────────────┤
│    子系统一：问答     │    子系统二：视频生成       │
│                      │                          │
│  RAG 管线             │  脚本 → 生图 → TTS → 合成  │
│  检索 → LLM → 回答    │                          │
└──────────────────────┴──────────────────────────┘
```

**前端入口**：Gradio（本机运行，两个 Tab 页——问答、生成视频）

---

## 3. 视频生成管线（MVP 方案 C）

### 3.1 管线流程

```
知识库 → LLM 写故事脚本 → 文生图(关键帧) → TTS 配音 → FFmpeg 合成 → 视频文件
```

对标**有声绘本**，静态插图 + 配音旁白 + 简单的缩放/平移动效（Ken Burns 效果），非逐帧动画。

### 3.2 各环节说明

| 环节 | 做什么 | 输入 | 输出 |
|------|--------|------|------|
| **脚本生成** | 从知识库检索适龄内容，LLM 改写为 2-3 分钟的故事脚本，分 5-8 个场景 | 年龄段、主题关键词、个性化参数（孩子名字/喜好） | 分镜脚本（每个场景的画面描述 + 旁白文字） |
| **关键帧生图** | 对每个场景描述调用文生图 API，生成风格统一的插图 | 场景描述 + 统一风格 prompt | 5-8 张插图 PNG |
| **TTS 配音** | 将旁白文字转为语音 | 分场景的旁白文字 | 单个或分段的 MP3 音频文件 |
| **视频合成** | FFmpeg 将图片+音频合成视频，每张图配缩放/平移动效 | 图片序列 + 音频 | MP4 视频文件 |

### 3.3 角色一致性策略

0-3 岁内容对角色一致性要求高。MVP 阶段采用以下策略：

- **每张生图都带上统一风格 prompt**（"cute illustration style for toddlers, soft pastel colors, simple shapes"）
- **场景描述中固定主角外观**（如"一只名叫[XX]的白色小兔子，长耳朵，穿蓝色背心"），每帧复用这个描述
- **不做角色参考图生图**（增加复杂度），用 prompt 工程来逼近一致性
- 如果一致性实在不够，降级为**无固定角色的科普/认知类内容**（动物、颜色、数字等），这些不需要角色一致性

## 4. 技术选型

| 层 | 选型 | 原因 |
|----|------|------|
| 整体框架 | Python | 整个 AI 生态都在 Python，一个语言搞定 |
| UI | Gradio | 零前端代码，两行搭出界面，本机跑 |
| RAG 引擎 | LlamaIndex 或手写 embedding + FAISS | 知识库规模小（几本书），不需要重型方案 |
| LLM | DeepSeek API 或通义千问 API | 国内访问稳定，中文能力强 |
| 文生图 | 通义万相 / 即梦 API | 国内可用，风格偏可爱适合幼儿内容 |
| TTS | Edge TTS（免费）或 Azure TTS | 声音自然，中文效果好 |
| 视频合成 | FFmpeg（subprocess 调用） | 成熟稳定，一张图+一段音频合成完全够用 |
| Embedding | text-embedding-3-small 或 bge-large-zh | 中文 embedding，价格便宜 |

## 5. 个性化参数

MVP 阶段支持的个性化维度：

| 参数 | 示例 | 来源 |
|------|------|------|
| 孩子名字 | "小月" | 用户输入 |
| 年龄 | "2 岁" | 用户输入 |
| 主题偏好 | "小兔子、星星、吃饭" | 用户输入或从对话历史提取 |
| 教育目标 | "培养刷牙习惯" | 知识库匹配 |

LLM 脚本生成时将这些参数注入 system prompt，故事角色、情节、对话围绕这些参数展开。

## 7. MVP 范围（第一期）

**必做：**
- [ ] 上传 2-3 本育儿书 PDF，构建知识库
- [ ] 教育问答 Tab：输入问题 → RAG 检索 → LLM 回答
- [ ] 视频生成 Tab：输入（孩子名字、年龄、主题）→ 脚本 → 生图 → TTS → 合成 MP4
- [ ] Gradio 界面，本机可运行

**不做：**
- 用户登录/权限系统
- 视频的 Web 播放器（下载到本地播放即可）
- 内容审核（人工审核，不搞自动）
- 模板系统（方案 B）
- 真实动画生成（方案 B）
- AI 视频生成/角色一致性方案（方案 A）
- 资源推荐/内容筛选功能
- 工作方向、生活方向

## 8. 已知风险

1. **文生图角色一致性**：多张图之间主角外观可能不统一。降级策略：首版优先做认知类内容（动物、颜色等无需固定角色）
2. **生成内容适龄性**：LLM 可能产出不适合 0-3 岁的内容。对策：脚本生成后人工扫一眼再进入生图环节
3. **生成耗时**：完整管线（脚本+5-8 张图+TTS+合成）可能需要 1-3 分钟，Gradio 页面需要 loading 状态
4. **Edge TTS 音色**：免费方案的中文童声效果有限，如果效果不好替换为 Azure TTS
